ESIG - Module 113 - Projet personnel

Objectif : Créer et publier un site web personnel en HTML/CSS démontrant la maîtrise des compétences fondamentales du développement web.
📋 Vue d'ensemble

🎯 Philosophie du projet

Plutôt que d'apprendre de manière passive, vous allez construire votre propre site web en découvrant et appliquant les technologies HTML et CSS. Ce projet vous permettra de développer votre autonomie, votre créativité et votre capacité à apprendre en faisant.

⚖️ Évaluation

  • Projet personnel 20 points sur 40 pour l'évaluation finale du module
  • Complété par un questionnaire écrit (20 points) sur des connaissances théoriques et des études de codes (correction, compréhension, complétion)

📅 Durée

  • 9 semaines de développement (semaines 4 à 12)
  • Points de suivi obligatoires avec l’enseignant toutes les 2 semaines

🎯 Objectifs d'apprentissage
  • Créer et structurer des pages web sémantiques avec HTML5

  • Créer un design responsive (mobile et desktop)
  • Gérer un projet avec Git et GitHub

  • Documenter et expliquer sa démarche

✅ Exigences techniques
🏗️ Structure HTML obligatoire

Votre site doit utiliser toutes ces balises HTML5 :

Structure sémantique :

  • <header> - En-tête du site/page
  • <nav> - Menu de navigation
  • <main> - Contenu principal
  • <section> - Sections de contenu
  • <article> - Articles/contenus autonomes (optionnel)
  • <footer> - Pied de page

Contenus textuels :

  • <h1>, <h2>, … - Hiérarchie des titres
  • <p> - Paragraphes
  • <ul>, <ol>, <li> - Listes
  • <a> - Liens hypertextes
  • <strong>, <em> - Emphases

Médias :

  • <img> - Images avec attributs alt
  • <figure>, <figcaption> - Figures avec légendes
🎨 Techniques CSS obligatoires

Votre site doit démontrer la maîtrise de :

Utilisez des variables CSS

  • Minimum requis :
    • 3 variables de couleurs
    • 2 variables de polices
    • 3 variables d'espacement

Mise en forme de base :

  • Typographie : font-family, font-size, font-weight, line-height
  • Couleurs : color, background-color
  • Espacement : margin, padding

Modèle de boîte :

  • width, height, max-width
  • border, border-radius
  • box-sizing: border-box

Positionnement :

  • display (block, inline, inline-block)
  • position (static, relative, absolute)
  • float OU Flexbox OU CSS Grid (au choix)

Design responsive :

  • @media queries (minimum 2 breakpoints)
  • Images adaptatives (max-width: 100%)
  • Navigation adaptée mobile/desktop

Interactivité :

  • :hover, :focus sur les liens
  • :visited pour les liens visités
  • Transitions CSS (transition) - optionnel mais apprécié
  • Filtres CSS (filter) - optionnel mais apprécié
📱 Exigences responsive
  • Mobile First : Site fonctionnel sur mobile (320px+)
  • Breakpoints : Adaptation tablette (768px+) et desktop (1024px+)
  • Navigation : Menu adapté aux petits écrans
  • Images : Redimensionnement automatique
🔧 Exigences techniques
  • Validation W3C : HTML et CSS sans erreurs
  • Tester la performance et l’expérience utilisateur (UX) avec PageSpeed Insights
    • Résultats pour mobile et bureau
  • Accessibilité : Attributs alt, structure logique, contrastes suffisants
  • Compatibilité : Fonctionne sur Chrome, Firefox, Safari
  • Polices : Toutes les polices doivent être téléchargées et hébergées dans le dossier fonts/ du projet
    • ❌ Interdiction d'utiliser des CDN externes (Google Fonts CDN, etc.)
    • ✅ Téléchargement et intégration locale avec @font-face
  • Images : Toutes les images doivent être hébergées dans le dossier img/ du projet
    • ❌ Pas de liens vers des images externes (Unsplash, Pixabay, etc.)
    • ✅ Images téléchargées et optimisées localement (< 1MB par image)
