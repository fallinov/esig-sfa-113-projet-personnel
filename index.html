  • Hébergé (GitHub Pages ou équivalent).

  • Design responsive mobile → desktop.
  • Polices locales dans /fonts (pas de CDN).
  • Images locales < 1 Mo dans /img.

README – structure attendue

  • Concept : thème, public, objectifs.
  • Fonctionnalités : navigation, sections, contenus.
  • Aperçu : capture dans img/screenshot.png.
  • Ressources : cours, docs, vidéos (liens).
  • Progression : jalons semaine par semaine.
  • Outils : IDE, navigateurs, Git, validateurs.
  • Design :
    • Palette : 3–5 couleurs avec hex + lien générateur.
    • Typographies : fichiers dans /fonts, déclaration @font-face, fallbacks.
  • Tests & validations :
    • HTML/CSS W3C : résultats.
    • PageSpeed mobile/bureau : Perf/Accessibilité/Best Practices/SEO /100.
    • Accessibilité : points WAVE corrigés.
    • Compatibilité : navigateurs + versions; appareils testés.
  • Transparence IA : outil, version, domaines d'aide (%), explications.
  • Bilan : acquis, difficultés/solutions, améliorations.

Structure HTML

  • <header>, <nav>, <main>, <section>, <footer>.
  • Titres hiérarchisés h1…h6.
  • <p>, <ul/ol/li>, <a>.
  • <strong>, <em>.
  • <img alt>, <figure>/<figcaption> si pertinent.

CSS requis

  • Variables : ≥3 couleurs, ≥2 polices, ≥3 espacements.
  • Typo : font-family/size/weight/line-height.
  • Couleurs, marges, paddings.
  • Boîte : width/height/max-width/border/radius/box-sizing.
  • Layout : display, position, Flexbox ou Grid (ou float).
  • Interactions : :hover, :focus, :visited.
  • Bonus : transition, filter.

Responsive & Qualité

  • Mobile-first 320 px+.
  • ≥2 @media (tablette ≥768, desktop ≥1024).
  • Images adaptatives max-width:100%.
  • Accessibilité : alt, contrastes, navigation clavier.
  • Compatibilité : Chrome 120+, Firefox 120+, Safari 16+ (ou versions testées).
  • Performance : audit PageSpeed mobile et desktop.
  • Pas de CDN (polices/images locales).

FAQ

Puis-je utiliser Bootstrap ou Tailwind ?

Non. Le but est de maîtriser le CSS natif.

Puis-je lier Google Fonts en CDN ?

Non. Télécharge et héberge les polices dans /fonts.

Combien de breakpoints ?

Au minimum deux : tablette et desktop.

Que mettre dans le README ?

Concept, ressources, progression, outils, tests, bilan, transparence IA.
