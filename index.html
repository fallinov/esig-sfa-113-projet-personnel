Projet personnel HTML5 + CSS3

Module 113 - Développement web front-end

Évaluation

20 pts projet
20 pts écrit

Durée

9 semaines

Suivi

Toutes les 2 semaines

Ce que tu dois faire :

  • Construire et publier un site en HTML5 + CSS3
  • Versionner et publier via GitHub (GitHub Pages)
  • Tenir un README.md réflexif
Voir les exigences Planning des jalons

Objectifs

Autonomie d'apprentissage

Capacité à chercher, sélectionner et assimiler de nouvelles connaissances

Créativité technique

Liberté d'expression dans le respect des contraintes techniques

Maîtrise pratique

HTML5, CSS3, Git/GitHub, responsive design

Compréhension profonde

100% du code produit doit être compris (y compris si généré par IA)

Méthodes professionnelles

Utilisation d'IDE (WebStorm), versioning, publication

À rendre

Livrable 1 · Site en ligne

  • Hébergé (GitHub Pages ou équivalent).
  • HTML/CSS valides W3C.
  • Design responsive mobile → desktop.
  • Polices locales dans /fonts (pas de CDN).
  • Images locales < 1 Mo dans /img.
  • Dépôt propre et structuré.

Arborescence minimale :

  • index.html
  • css/
  • fonts/
  • img/
  • README.md

Livrable 2 · README.md

  • Concept, public cible, objectifs.
  • Fonctionnalités et sections du site.
  • Capture d'écran du site.
  • Ressources et progression.
  • Outils et méthodologie (mobile-first, commits).
  • Tests et validations.
  • Bilan, difficultés, améliorations.
  • Transparence sur l'usage d'IA.

Exigences

Structure HTML

  • <header>, <nav>, <main>, <section>, <footer>.
  • Titres hiérarchisés h1…h6.
  • <p>, <ul/ol/li>, <a>.
  • <strong>, <em>.
  • <img alt>, <figure>/<figcaption> si pertinent.

CSS requis

  • Variables : ≥3 couleurs, ≥2 polices, ≥3 espacements.
  • Typo : font-family/size/weight/line-height.
  • Couleurs, marges, paddings.
  • Boîte : width/height/max-width/border/radius/box-sizing.
  • Layout : display, position, Flexbox ou Grid (ou float).
  • Interactions : :hover, :focus, :visited.
  • Bonus : transition, filter.

Responsive & Qualité

  • Mobile-first 320 px+.
  • ≥2 @media (tablette ≥768, desktop ≥1024).
  • Images adaptatives max-width:100%.
  • Accessibilité : alt, contrastes, navigation clavier.
  • Compatibilité : Chrome 120+, Firefox 120+, Safari 16+ (ou versions testées).
  • Performance : audit PageSpeed mobile et desktop.
  • Pas de CDN (polices/images locales).

Jalons

Phase 1 · Fondations (Sem. 1–3)

  • Comptes GitHub et JetBrains créés.
  • Premier "Hello World" publié (GitHub Pages).
  • Config IDE (WebStorm/VS Code).
  • Choix du sujet et de la plateforme d'apprentissage.
  • Palette de couleurs + polices locales.

Phase 2 · Développement (Sem. 4–10)

  1. S4 – Jalon 1 : Concept validé + structure de base.
  2. S6 – Jalon 2 : HTML complet + navigation. (Suivi)
  3. S8 – Jalon 3 : CSS de base + début responsive. (Suivi)
  4. S10 – Jalon 4 : Responsive fonctionnel + tests multi-devices. (Suivi)

Phase 3 · Finalisation (Sem. 11–12)

  • S11 – Optimisation : W3C OK, images optimisées, audits perf.
  • S12 – Livraison : Site publié + README complet. (Suivi final)

Outils

Outils de travail

  • IDE : WebStorm (étudiants) ou VS Code.
  • Versioning : Git + GitHub.
  • Navigateurs : Chrome DevTools, Firefox, Safari.
  • Typo : Google Web Fonts Helper (téléchargement local).
  • Couleurs : Coolors, Adobe Color.

Validation & tests

  • HTML : W3C Markup Validator / Nu Html Checker.
  • CSS : W3C CSS Validator.
  • Accessibilité : WAVE.
  • Performance/UX : PageSpeed Insights.
  • Responsive : outils intégrés navigateurs.
  • Compatibilité : Chrome 120+, Firefox 120+, Safari 16+ (ou versions testées).

Validation

Grille d'évaluation (20 pts)

  • HTML/CSS : validité, propreté, sémantique – 4 pts
  • Responsive : mobile-first, breakpoints, navigation – 4 pts

  • Code : variables, organisation, commentaires – 3 pts
  • Performance : optimisations, audits – 3 pts
  • Bonus : créativité, fonctionnalités avancées – 2 pts

IA

Utilisation de l'IA autorisée

L'utilisation d'outils d'IA (ChatGPT, GitHub Copilot, etc.) est autorisée sous certaines conditions :

  • Compréhension obligatoire : Vous devez comprendre 100% du code généré
  • Transparence : Mentionnez l'usage d'IA dans votre README
  • Personnalisation : Adaptez le code à votre projet
  • Apprentissage : Utilisez l'IA comme outil d'apprentissage
  • Validation : Testez et validez tout code généré

FAQ

Questions fréquentes

Puis-je utiliser un framework CSS comme Bootstrap ?

Non, ce projet vise à maîtriser le CSS pur. Utilisez Flexbox ou Grid pour vos layouts.

Combien de pages doit contenir mon site ?

Minimum 3 pages : accueil, contenu principal, et une page supplémentaire de votre choix.

Puis-je utiliser JavaScript ?

Le JavaScript est optionnel. Concentrez-vous d'abord sur HTML/CSS.

Comment optimiser mes images ?

Utilisez des outils comme TinyPNG ou ImageOptim. Format WebP recommandé.
