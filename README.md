# 🚀 Module 113 - Site Web Projet Personnel HTML/CSS

## 📋 Description du projet

Ce site web présente le **Module 113 - Bases du développement web (HTML & CSS)** de l'ESIG (École Supérieure d'Informatique de Gestion) en Suisse. Il s'agit d'un exemple concret de ce que les étudiants peuvent réaliser en respectant toutes les bonnes pratiques du développement web moderne.

## 🎯 Objectifs du site

- **Présenter le projet personnel** aux étudiants de manière claire et engageante
- **Servir d'exemple** de bonnes pratiques HTML5, CSS3 et JavaScript
- **Démontrer** un design responsive et accessible
- **Inspirer** les étudiants par la qualité du code et du design

## 🛠️ Technologies utilisées

### Frontend
- **HTML5** : Structure sémantique complète
- **CSS3** : Variables CSS, Flexbox, Grid, animations
- **JavaScript ES6+** : Navigation mobile, animations au scroll
- **Responsive Design** : Mobile-first approach

### Outils de développement
- **WebStorm** : IDE principal
- **Git & GitHub** : Versioning et déploiement
- **Chrome DevTools** : Debugging et optimisation

## 📁 Structure du projet

```
esig-sfa-113-projet-personnel/
```

## 🎨 Design et UX

### Palette de couleurs
- **Couleur principale** : `#667eea` (Bleu moderne)
- **Couleur secondaire** : `#764ba2` (Violet)
- **Couleur d'accent** : `#f59e0b` (Orange)
- **Couleurs neutres** : Échelle de gris de 50 à 900

### Typographie
- **Police principale** : Inter (corps de texte)
- **Police des titres** : Poppins (headings)
- **Police monospace** : JetBrains Mono (code)

### Variables CSS
Le projet utilise un système complet de variables CSS pour :
- **Couleurs** : Palette cohérente et maintenable
- **Espacements** : Système d'échelle harmonieux
- **Typographie** : Tailles et poids standardisés
- **Breakpoints** : Responsive design structuré

## 📱 Responsive Design

### Breakpoints
- **Mobile** : < 640px (SM)
- **Tablette** : 640px - 768px (MD)
- **Desktop** : 768px - 1024px (LG)
- **Large** : > 1024px (XL)

### Approche Mobile-First
- Design optimisé pour mobile en premier
- Progressive enhancement pour les écrans plus grands
- Navigation adaptée avec menu hamburger

## ♿ Accessibilité

### Standards respectés
- **WCAG 2.1** : Niveau AA
- **Navigation au clavier** : Complète
- **Lecteurs d'écran** : Compatible
- **Contrastes** : Respect des ratios recommandés

### Fonctionnalités d'accessibilité
- Skip links pour la navigation
- Attributs ARIA appropriés
- Focus visible et géré
- Support des préférences utilisateur (réduction de mouvement)

## 🚀 Performance

### Optimisations
- **Polices** : Chargement optimisé avec `font-display: swap`
- **Images** : Lazy loading avec Intersection Observer
- **CSS** : Variables et sélecteurs optimisés
- **JavaScript** : Code modulaire et efficace

### Métriques cibles
- **Lighthouse Score** : > 90/100
- **First Contentful Paint** : < 1.5s
- **Largest Contentful Paint** : < 2.5s
- **Cumulative Layout Shift** : < 0.1

## 🔧 Installation et développement

### Prérequis
- Navigateur moderne (Chrome, Firefox, Safari, Edge)
- Serveur web local (optionnel)

### Démarrage rapide
1. Cloner le repository
2. Ouvrir `index.html` dans un navigateur
3. Ou utiliser un serveur local :
   ```bash
   # Avec Python
   python -m http.server 8000
   
   # Avec Node.js
   npx serve .
   ```
